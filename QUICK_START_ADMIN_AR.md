# 🔐 بيانات تسجيل الدخول إلى لوحة تحكم المشرف

## ✅ البيانات الافتراضية

بعد إنشاء حساب المشرف الأول، استخدم البيانات التالية:

```
┌─────────────────────────────────────┐
│  اسم المستخدم: admin                │
│  كلمة المرور: Admin@123456          │
│  الاسم: مدير النظام                  │
│  الصلاحية: مسؤول رئيسي              │
└─────────────────────────────────────┘
```

⚠️ **مهم جداً:** غيّر كلمة المرور فوراً بعد أول دخول!

---

## 📍 رابط لوحة التحكم

```
http://localhost:3001/admin/
```

---

## 🚀 كيفية إنشاء حساب المشرف الأول

### الطريقة الأسهل: استخدام SQL في Supabase

1. **افتح Supabase Dashboard** لمشروعك

2. **اذهب إلى SQL Editor** (من القائمة الجانبية)

3. **انسخ والصق الكود التالي:**

```sql
INSERT INTO admins (username, password, name, role) VALUES (
  'admin',
  '$2a$10$rN8VqhK5JzKxH9L2mP3wOeYtR6sU7vW8xZ9aB1cD2eF3gH4iJ5kL6',
  'مدير النظام',
  'super_admin'
);
```

4. **اضغط "Run"** لتنفيذ الأمر

5. **تم!** ✅ يمكنك الآن تسجيل الدخول

---

## 📋 خطوات تسجيل الدخول

1. ✅ تأكد من تشغيل الخادم (Backend):
   ```bash
   cd backend
   npm start
   ```

2. ✅ افتح المتصفح واذهب إلى:
   ```
   http://localhost:3001/admin/
   ```

3. ✅ أدخل البيانات:
   - **اسم المستخدم:** `admin`
   - **كلمة المرور:** `Admin@123456`

4. ✅ اضغط **"دخول"**

5. ✅ ستفتح لوحة التحكم! 🎉

---

## 🛡️ تغيير كلمة المرور

بعد أول دخول:

1. في لوحة التحكم، اذهب إلى **الإعدادات** ⚙️
2. اختر **"تغيير كلمة المرور"**
3. أدخل:
   - كلمة المرور الحالية: `Admin@123456`
   - كلمة المرور الجديدة: (اختر كلمة قوية)
   - تأكيد كلمة المرور
4. اضغط **حفظ**

---

## 👥 إضافة مديرين جدد

بصفتك **مسؤول رئيسي**، يمكنك إضافة مديرين:

1. اذهب إلى **"إدارة المديرين"** 👥
2. اضغط **"إضافة مدير جديد"**
3. أدخل البيانات:
   - الاسم الكامل
   - اسم المستخدم (فريد)
   - كلمة المرور
   - الدور:
     - **مسؤول (Admin):** صلاحيات عادية
     - **مسؤول رئيسي (Super Admin):** صلاحيات كاملة
4. اضغط **حفظ**

---

## 🔑 مواصفات كلمة المرور القوية

✅ 8 أحرف على الأقل  
✅ أحرف كبيرة وصغيرة (A-Z, a-z)  
✅ أرقام (0-9)  
✅ رموز خاصة (!@#$%^&*)  

**أمثلة:**
- `Musharaka@2024!Secure`
- `Admin#Safe$99`
- `Str0ng&P@ss!`

## ❓ حل المشاكل الشائعة

### المشكلة: "بيانات الدخول غير صحيحة"

**الحل:**
1. تأكد من تنفيذ أمر SQL بنجاح في Supabase
2. تحقق من كتابة البيانات بشكل صحيح
3. اسم المستخدم وكلمة المرور حساسان لحالة الأحرف

---

### المشكلة: "لا يمكن الوصول للوحة التحكم"

**الحل:**
1. تأكد من تشغيل الخادم:
   ```bash
   cd backend
   npm start
   ```
2. يجب أن ترى رسالة:
   ```
   ✓ Supabase client initialized
   
   ╔══════════════════════════════════════╗
   ║   Musharaka Backend API              ║
   ║   http://localhost:3001              ║
   ╚══════════════════════════════════════╝
   ```

---

### المشكلة: "خطأ في قاعدة البيانات"

**الحل:**
1. تحقق من ملف `backend/.env`:
   ```env
   SUPABASE_URL=https://xxxxx.supabase.co
   SUPABASE_SERVICE_ROLE_KEY=eyJhbGc...
   ```
2. تأكد من صحة البيانات من Supabase Dashboard
3. تحقق من وجود جدول `admins` في قاعدة البيانات

---

### المشكلة: نسيت كلمة المرور

**الحل 1 - عبر SQL:**
```sql
-- إعادة تعيين كلمة المرور لـ admin
UPDATE admins 
SET password = '$2a$10$rN8VqhK5JzKxH9L2mP3wOeYtR6sU7vW8xZ9aB1cD2eF3gH4iJ5kL6'
WHERE username = 'admin';
```

**الحل 2 - إنشاء مدير جديد:**
```sql
INSERT INTO admins (username, password, name, role) VALUES (
  'newadmin',
  '$2a$10$rN8VqhK5JzKxH9L2mP3wOeYtR6sU7vW8xZ9aB1cD2eF3gH4iJ5kL6',
  'مدير جديد',
  'super_admin'
);
```

## 📊 أنواع الصلاحيات

### 👑 مسؤول رئيسي (Super Admin)
- ✅ إدارة الطلبات
- ✅ إدارة الرسائل

- ✅ تعديل الإعدادات

- ✅ إدارة الرسائل
- ✅ عرض الإعدادات
- ✅ عرض سجل النشاطات
